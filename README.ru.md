# BiosGuard 

> **Инструмент верификации целостности UEFI-прошивки** — Решение для предзагрузочной безопасности, обнаруживающее несанкционированные модификации BIOS с использованием криптографического хеширования SHA-256.

[English version](README.md)

---

## Обзор

**BiosGuard** — это EFI-приложение, выполняющееся на уровне прошивки (перед загрузкой ОС) для проверки целостности компонентов BIOS. Оно обеспечивает криптографическую гарантию того, что критические тома прошивки не были изменены.

### Ключевые возможности

- **Криптографическая верификация** — хеширование SHA-256 для томов прошивки
- **Предзагрузочное выполнение** — работает на уровне UEFI, независимо от ОС
- **Парсинг томов прошивки** — находит и анализирует UEFI PI-тома по GUID
- **База хешей** — встроенный whitelist доверенных хешей прошивки
- **Аудит** — генерация отчётов о целостности и записей неизвестных хешей
- **Легковесность** — оптимизировано для минимального потребления памяти

## Требования

### Для сборки

- [Visual Studio 2022](https://visualstudio.microsoft.com/) (v17.0+)
- [EDK2](https://github.com/tianocore/edk2) (UEFI Development Kit II)
- Windows SDK 10.0 или новее
- 64-битная UEFI-прошивка (целевая)

### Для выполнения

- x86_64 UEFI-система (без поддержки Legacy BIOS)
- SMBIOS 3.0+ для определения размера BIOS
- Доступ к UEFI Shell
- Отключенный Secure Boot (для неподписанных сборок)

## Быстрый старт

### 1. Клонирование репозитория

```bash
git clone https://github.com/username/BiosGuard.git
cd BiosGuard
```

### 2. Настройка окружения сборки

Установите переменную окружения `EDK2_PATH`:

```powershell
# PowerShell (от Администратора)
[Environment]::SetEnvironmentVariable("EDK2_PATH", "C:\Workspace\edk2", "Machine")

# Или Command Prompt
setx EDK2_PATH "C:\Workspace\edk2" /M
```

### 3. Сборка

Откройте `BiosGuard.sln` в Visual Studio:

```powershell
BiosGuard.sln
```

Выберите конфигурацию `Release|x64` и соберите проект (`Ctrl+Shift+B`).

Результат: `bin/BiosGuard.efi`

### 4. Запуск

Скопируйте `BiosGuard.efi` на USB-накопитель FAT32 и загрузитесь в UEFI Shell:

```shell
FS0:
BiosGuard.efi
```

## Обновление базы хешей

После первого запуска новые хеши прошивки добавляются в `hash.bin`. Для встраивания:

```bash
cd tools
python3 HashAdder.py -i hash.bin -o HashBase.h
```

Пересоберите проект для включения обновлённых хешей.

## Вопросы безопасности

> **Внимание**: Этот инструмент работает на уровне привилегий Ring -1.

- **Не панацея** — Защищает от модификаций прошивки, но не от аппаратных атак
- **Secure Boot** — Требует выполнения неподписанного EFI; подпишите собственными ключами для production
- **Покрытие хешами** — Проверяет только тома с PEI/DXE-модулями; NVRAM исключён по дизайну
- **Физический доступ** — Злоумышленник с физическим доступом может обойти программную защиту

### Модель угроз

| Угроза | Защита | Статус |
|--------|--------|--------|
| BIOS-вредоносное ПО (руткит) | SHA-256 верификация | ✅ Защищено |
| Даунгрейд прошивки | Хеши с учётом версий | ⚠️ Требует обновления |
| Атака "Злая горничная" | Предзагрузочная верификация | ⚠️ Зависит от оборудования |
| Атака на цепочку поставок | Подписи производителя | ❌ Не покрывается |
